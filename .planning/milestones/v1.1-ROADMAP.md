# Milestone v1.1: Local STT

**Status:** SHIPPED 2026-03-02
**Phases:** 4-6
**Total Plans:** 3

## Overview

Replace whisper CLI subprocess with HTTP calls to existing mlx-audio server for GPU-accelerated transcription on Apple Silicon. Adds local STT client, backend selection, model configuration, and enriched transcript metadata.

## Phases

### Phase 4: Local STT Client

**Goal**: HTTP client calling /v1/audio/transcriptions on existing mlx-audio server, response parsing, error classification for STT failure modes
**Depends on**: v1.0 complete (server lifecycle already managed)
**Plans**: 1 plan

Plans:
- [x] 04-01: LocalSTT HTTP client with multipart upload, response parsing, error classification

**Details:**
- LocalSTTError value type with 7 classified failure modes (server_down, stale_pid, oom, timeout, port_conflict, unsupported_format, model_error)
- STT-specific HTTP error and response classifiers mirroring TTS patterns
- LocalSTT function with multipart form upload, server liveness validation, and response parsing
- verbose_json response format for segments with timestamps and duration
- Default STT model: mlx-community/whisper-large-v3-turbo
- Language hardcoded to "zh" matching project scope

### Phase 5: STT Backend Selection

**Goal**: Backend selection flag for STT (local vs whisper), backward compatibility with whisper CLI, process-voice pipeline update
**Depends on**: Phase 4
**Plans**: 1 plan

Plans:
- [x] 05-01: STT backend selection with CLI command and ProcessVoice routing

**Details:**
- `audio stt --in <path> [--backend local|whisper] --json` command registered in CLI router
- Backend resolution chain: flag > config (stt_backend) > env (XUEZH_AUDIO_STT_BACKEND) > default "whisper"
- ProcessVoice routes STT through resolved backend instead of hardcoding whisper
- LocalSTTError, ToolMissingError, and ProcessFailedError all emit structured error envelopes

### Phase 6: STT Polish

**Goal**: Model selection, transcript format improvements, confidence/timestamp metadata
**Depends on**: Phase 5
**Plans**: 1 plan

Plans:
- [x] 06-01: STT model selection via --model flag and enriched transcript metadata

**Details:**
- STT model selectable via `audio stt --model` flag with resolution chain (flag > config > env > default)
- ProcessVoice accepts and threads sttModel to LocalSTT when using local backend
- extractTranscript enriched with avg_logprob, no_speech_prob, compression_ratio per segment
- Word-level timestamps (word, start, end, probability) extracted per segment when available
- Duration extracted at top level by extractTranscript, benefiting both local and whisper backends

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- LocalSTTError mirrors LocalTTSError as value type with same BACKEND_FAILED:reason:message format
- Multipart form upload with file/model/language/response_format fields for OpenAI-compatible API
- verbose_json response format for segments with timestamps and duration
- STT backend default is "whisper" (preserves existing behavior, STT-04)
- ProcessVoice sttBackend resolved independently from process_voice_backend
- Model resolution uses simple inline chain (not resolveAudioBackend which has global fallback)
- All transcript metadata fields (confidence, words, duration) are additive — omitted when absent

**Issues Resolved:**
- Removed redundant duration extraction from LocalSTT when extractTranscript was enhanced (auto-fixed in Phase 6)

**Issues Deferred:** None

**Technical Debt Incurred:** None
