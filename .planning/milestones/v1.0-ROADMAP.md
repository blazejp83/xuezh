# Milestone v1.0: Local TTS

**Status:** SHIPPED 2026-03-01
**Phases:** 1-3
**Total Plans:** 6

## Overview

Add local mlx-audio TTS support to xuezh for Apple Silicon. Server lifecycle management, HTTP TTS client, backend selection, and speech style control via Qwen3-TTS.

## Phases

### Phase 1: Server Lifecycle

**Goal**: Agent can start, stop, and query the mlx-audio TTS server via CLI commands with reliable process management
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01: Server spawn with process group isolation, PID/port tracking, health polling, warm-up
- [x] 01-02: Server stop with clean process group shutdown, status command

### Phase 2: Local TTS Backend

**Goal**: Agent can generate Chinese speech via HTTP calls to running mlx-audio server
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:
- [x] 02-01: HTTP client calling /v1/audio/speech, voice selection, ffmpeg conversion
- [x] 02-02: Structured JSON error handling for all 6 failure modes

### Phase 3: Polish & Integration

**Goal**: Edge-tts fallback preserved, backend selection flag, instruct parameter, dialect voices
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:
- [x] 03-01: Backend selection flag, edge-tts backward compatibility
- [x] 03-02: Instruct parameter for speech style, dialect voice support

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Keep Go codebase, don't rewrite (90% of code unrelated to TTS)
- Qwen3-TTS via mlx-audio for best Chinese quality + Apple Silicon optimization
- HTTP localhost for TTS communication (OpenAI-compatible server)
- Server lifecycle in CLI (agent needs single interface)
- Keep edge-tts as fallback backend
- LocalTTSError as value type (errors.As matches directly)
- Three-phase shutdown: SIGTERM → 5s wait → SIGKILL → 2s wait
- Default voice "Vivian" for local backend

**Issues Deferred:** None
