  <name>Task 1: Create LocalTTS function in audio/local.go</name>
  <files>internal/xuezh/audio/local.go</files>
  <action>
Create new file `internal/xuezh/audio/local.go` in the audio package with a `LocalTTS` function.

**Function signature:** `func LocalTTS(text, voice, outPath, purpose string) (AudioResult, error)`

**Implementation:**
1. Read port from state file via `readPortFile()`. If 0, return error "server not running".
2. Read model from state file via `readModelFile()`.
3. Read PID via `readPIDFile()` and check `processAlive(pid)`. If dead, call `cleanStateFiles()` and return error "server not running (stale PID)".
4. Default voice to "Vivian" if empty string passed.
5. Resolve outPath via `paths.ResolveInWorkspace(outPath)` and create parent dirs.
6. Ensure ffmpeg available via `process.EnsureTool("ffmpeg")`.
7. Create temp file path for raw server response: `.local-tts-{uuid}.wav` in same directory as outPath.
8. HTTP POST to `http://127.0.0.1:{port}/v1/audio/speech` with JSON body `{"model": "{model}", "input": "{text}", "voice": "{voice}"}`. Use Content-Type: application/json. Timeout: 120 seconds (long texts need time).
9. If HTTP error or non-200 status, return error with response body context.
10. Write response body to temp WAV file, defer cleanup.
11. Determine output format from outPath extension (default "ogg" like TTSAudio does).
12. Convert temp WAV to target format via `buildConvertCommand()` + `process.RunChecked()`.
13. Build artifact via `artifactFor()`.
14. Return AudioResult with data map including: text, voice, out, model, backend: {id: "local", features: ["tts"]}.

**Imports needed:** bytes, fmt, io, net/http, os, path/filepath, strings, time, github.com/google/uuid, plus internal packages (paths, process, envelope).

**Follow existing patterns from TTSAudio in audio.go** — same temp file cleanup, same artifact construction, same output format detection.
