Create `internal/xuezh/audio/local_stt.go` with:

1. **LocalSTTError** — value type (struct, not pointer) mirroring LocalTTSError exactly:
   - `Reason string` — "server_down", "stale_pid", "oom", "timeout", "port_conflict", "unsupported_format", "model_error"
   - `Message string` — human-readable description
   - `Details map[string]any` — structured details for error envelope
   - `Error() string` returns `"BACKEND_FAILED:{reason}:{message}"`

2. **classifySTTHTTPError(err error, port, pid int) LocalSTTError** — mirrors classifyHTTPError from local.go:
   - Timeout → reason "timeout"
   - Connection refused + live PID → "port_conflict"
   - Connection refused + dead PID → "stale_pid" (call cleanStateFiles)
   - Connection reset → "oom"
   - Fallback → "server_down"
   Reuse existing `isTimeout`, `isConnectionRefused`, `isConnectionReset` helpers from local.go (same package, already accessible).

3. **classifySTTHTTPResponse(statusCode int, respBody []byte, port int) LocalSTTError** — mirrors classifyHTTPResponse but STT-specific:
   - HTTP 400/415/422 with "unsupported" or "format" or "invalid" → reason "unsupported_format"
   - HTTP 500 with "model" or "load" or "not found" → reason "model_error"
   - HTTP 500 with "memory" or "oom" or "killed" → reason "oom"
   - Default → "server_down" with full details
   Body preview capped at 1KB (same as TTS pattern).

4. **Default STT model constant**: `defaultSTTModel = "mlx-community/whisper-large-v3-turbo"` — Phase 6 will make this configurable.

<task type="auto">
  <name>Task 2: Create LocalSTT function with multipart HTTP client and response parsing</name>
  <files>internal/xuezh/audio/local_stt.go</files>
  <action>
Add `LocalSTT(inPath, model string) (SttResult, error)` to `local_stt.go`:

1. **Server liveness check** (same as LocalTTS steps 1-3):
   - Read port from `readPortFile()`. If 0 → LocalSTTError{Reason:"server_down"}
   - Read PID from `readPIDFile()`. If 0 → clean + LocalSTTError{Reason:"server_down"}
   - Check `processAlive(pid)`. If false → clean + LocalSTTError{Reason:"stale_pid"}

2. **Model resolution**: If `model == ""`, use `defaultSTTModel`.

3. **Input file validation**: `expandHome(inPath)`, then `os.Stat()`. If not found → return regular error (not LocalSTTError — same pattern as LocalTTS for non-server errors).

4. **Multipart form creation** using `mime/multipart`:
   - Create `bytes.Buffer` + `multipart.NewWriter(buf)`
   - Add file field: `writer.CreateFormFile("file", filepath.Base(inputPath))` → copy file contents
   - Add text fields: `writer.WriteField("model", model)`, `writer.WriteField("language", "zh")`, `writer.WriteField("response_format", "verbose_json")`
   - Close writer

5. **HTTP POST** to `http://127.0.0.1:{port}/v1/audio/transcriptions`:
   - Content-Type: `writer.FormDataContentType()`
   - Timeout: 120s (same as TTS)
   - On error → `classifySTTHTTPError(err, port, pid)`
   - On non-200 → read body (limited 1KB), `classifySTTHTTPResponse(statusCode, body, port)`

6. **Parse response JSON**:
   - Unmarshal into `map[string]any`
   - Extract transcript using same structure as `extractTranscript()`: `{text, segments[{start, end, text}], language}`
   - The verbose_json response has `{text, language, duration, segments[{id, seek, start, end, text, ...}]}` — extract only `start`, `end`, `text` per segment (matching existing format)
   - Add `duration` to transcript if present

7. **Write transcript artifact** (same pattern as STTAudio):
   - `artifactPath("stt-local-{basename}", "json", now)`
   - Write transcript JSON to file
   - Create artifact with MIME "application/json", purpose "transcript"

8. **Return SttResult**:
   - Data: `{"in": inputPath, "model": model, "backend": {"id": "local", "features": ["stt"]}, "transcript": transcript}`
   - Artifacts: `[transcriptArtifact]`
   - Truncated: false, Limits: {}
  </action>
