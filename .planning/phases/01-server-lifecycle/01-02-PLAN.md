**StopResult type:**
```go
type StopResult struct {
    Status     string // "stopped"
    WasRunning bool   // true if a server was actually running and was stopped
}
```

**StatusResult type:**
```go
type StatusResult struct {
    Status        string  // "running" or "stopped"
    Port          int     // 0 if stopped
    PID           int     // 0 if stopped
    Model         string  // "" if stopped
    UptimeSeconds float64 // 0 if stopped
}
```

**StopServer() (StopResult, error):**

1. Read PID from pidPath(). If file doesn't exist, return `StopResult{Status: "stopped", WasRunning: false}` — no error, the agent gets a clean response.

2. Parse PID as int. Check if process alive via `syscall.Kill(pid, 0)`:
   - If ESRCH (process dead): clean up files (PID, port, model), return `StopResult{Status: "stopped", WasRunning: false}`
   - If alive: proceed to shutdown

3. **Three-phase graceful shutdown:**
   - Phase 1: Send `syscall.Kill(-pid, syscall.SIGTERM)` (negative PID = process GROUP). This ensures uvicorn workers and Metal shader subprocesses are included.
   - Wait up to 5 seconds, polling `syscall.Kill(pid, 0)` every 250ms. If ESRCH → process is dead, proceed to cleanup.
   - Phase 2: If still alive after 5s, send `syscall.Kill(-pid, syscall.SIGKILL)` to force-kill the entire process group. Wait up to 2 seconds for confirmation.
   - Phase 3: Remove PID, port, and model files via `os.Remove()`. Do NOT remove server.log (keep for post-mortem debugging).

4. Return `StopResult{Status: "stopped", WasRunning: true}`.

**Important for SIGTERM/SIGKILL:**
- Use negative PID (`-pid`) to kill the process GROUP, not just the leader. This is critical for preventing zombie uvicorn workers.
- Handle `syscall.Kill` errors gracefully — ESRCH means already dead (success), EPERM means permission denied (unlikely for our own process, but return error).

**ServerStatus() (StatusResult, error):**

1. Read PID from pidPath(). If file doesn't exist, return `StatusResult{Status: "stopped"}`.

2. Parse PID. Check alive via `syscall.Kill(pid, 0)`:
   - If ESRCH: clean up stale files, return `StatusResult{Status: "stopped"}`.

3. Read port from portPath(), model from modelPath().

4. **Verify server is actually responsive** (not just a PID-recycled unrelated process): Make a quick `GET http://127.0.0.1:{port}/v1/models` with 2s timeout.
   - If 200 OK: server is running and responsive.
   - If error/timeout: process exists but server is unresponsive. Report `Status: "running"` but include a note. The agent can decide to stop and restart.

5. **Calculate uptime:** Get the modification time of the PID file (`os.Stat(pidPath)`). Uptime = `time.Since(modTime).Seconds()`. This is approximate but sufficient — the PID file is written at spawn time.

6. Return `StatusResult{Status: "running", Port: port, PID: pid, Model: model, UptimeSeconds: uptime}`.

**Helper: isProcessAlive(pid int) bool:**
```go
func isProcessAlive(pid int) bool {
    err := syscall.Kill(pid, 0)
    return err == nil
}
```

**Important:**
- All file reads should handle missing files gracefully (os.IsNotExist → return "stopped" status)
- Parse port/PID with `strconv.Atoi` and handle parse errors
- Don't import new external packages — stdlib only

```go
func runAudioServerStatus(args []string) int {
    fs := flag.NewFlagSet("audio server status", flag.ContinueOnError)
    fs.SetOutput(os.Stderr)
    _ = fs.Bool("json", true, "Output JSON envelope")
    if err := fs.Parse(args); err != nil {
        return 1
    }
    result, err := audio.ServerStatus()
    if err != nil {
        return emitTypedError("audio.server.status", "BACKEND_FAILED", err.Error(), nil)
    }
    data := map[string]any{
        "status": result.Status,
    }
    if result.Status == "running" {
        data["port"] = result.Port
        data["pid"] = result.PID
        data["model"] = result.Model
        data["uptime_seconds"] = result.UptimeSeconds
    }
    out := envelope.OK("audio.server.status", data, nil, false, nil)
    return emit(out)
}
```

When status is "stopped", only include `status` key (no port/pid/model/uptime). When "running", include all fields.

- Follow existing patterns exactly — look at runAudioConvert for error handling structure
- Both commands should NEVER return exit code 1 for "no server running" — that's a valid status, not an error. Only return 1 for actual failures (permission denied, corrupted state files, etc.)
