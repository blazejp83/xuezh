Create `internal/xuezh/audio/server.go` in the existing `audio` package with server lifecycle management.

**Types:**

```go
type ServerState struct {
    Status         string  // "ready", "starting", "stopped", "error"
    Port           int
    PID            int
    Model          string
    StartupSeconds float64 // time from spawn to ready
}
```

**Helper functions for file paths (use paths.EnsureWorkspace()):**
- `serverDir() (string, error)` — returns `{workspace}/server/`, creates if needed
- `pidPath() (string, error)` — returns `{serverDir}/pid`
- `portPath() (string, error)` — returns `{serverDir}/port`
- `modelPath() (string, error)` — returns `{serverDir}/model`
- `logPath() (string, error)` — returns `{serverDir}/server.log`

**StartServer(port int, model string) (ServerState, error):**

Implementation steps:

1. **Clean stale state:** Call `cleanStaleServer()` — reads PID file, checks if process alive via `syscall.Kill(pid, 0)`. If ESRCH (dead), remove PID/port/model files. If alive, try GET `http://127.0.0.1:{port}/v1/models` — if responds, return error "server already running". If no response, kill the stale process group with `syscall.Kill(-pid, syscall.SIGKILL)` and clean files.

2. **Check port availability:** Use `net.Listen("tcp", fmt.Sprintf("127.0.0.1:%d", port))`. If error, return BACKEND_FAILED error with details `{"reason": "port_conflict", "port": port}`. Close listener immediately after success.

3. **Spawn subprocess:**
   ```go
   cmd := exec.Command("python3", "-m", "mlx_audio.server",
       "--host", "127.0.0.1",
       "--port", strconv.Itoa(port),
   )
   cmd.SysProcAttr = &syscall.SysProcAttr{Setpgid: true}
   // Redirect stdout/stderr to log file (O_CREATE|O_WRONLY|O_TRUNC — truncate on restart)
   logFile, _ := os.OpenFile(logPath, os.O_CREATE|os.O_WRONLY|os.O_TRUNC, 0644)
   cmd.Stdout = logFile
   cmd.Stderr = logFile
   cmd.Start() // non-blocking
   ```
   Do NOT call `cmd.Wait()` — the server must outlive the CLI process. Close the logFile handle after Start() returns (the child process has its own fd).

4. **Write state files:** Write PID (`cmd.Process.Pid`), port, and model as plain text integers/strings to their respective files. Use `os.WriteFile` with 0644 permissions.

5. **Health polling (waitForReady):**
   - Create `http.Client{Timeout: 2 * time.Second}`
   - Poll `GET http://127.0.0.1:{port}/v1/models` every 250ms
   - Overall timeout: 120 seconds
   - On each tick: if HTTP GET returns 200, server is listening — proceed to warm-up
   - If timeout exceeded, kill the process group, clean files, return BACKEND_FAILED error with `{"reason": "timeout", "timeout_seconds": 120}`
   - Between polls, check if process is still alive (`syscall.Kill(pid, 0)`). If dead, read last 20 lines of server.log, return BACKEND_FAILED with `{"reason": "crash", "log_tail": "..."}`

6. **Warm-up TTS request:**
   After health check passes, send a warm-up POST to `/v1/audio/speech`:
   ```json
   {"model": "{model}", "input": "你好", "voice": "Vivian"}
   ```
   Content-Type: application/json. Use `http.Client{Timeout: 60 * time.Second}` (model loading can take 30-40s on first request).
   - If 200: discard response body, model is warm
   - If error or non-200: don't fail the start — log a warning but still return "ready" (the model may warm up on the next real request)

7. **Return ServerState** with Status="ready", Port, PID, Model, and StartupSeconds (measured from step 3 to step 6 completion).

**Important implementation notes:**
- Import `net/http`, `syscall`, `net`, `strconv`, `time`, `io`, `bufio` as needed
- Do NOT import any external packages — use only stdlib + existing internal packages
- The `python3` binary must be on PATH — use `process.EnsureTool("python3")` before spawning
- Use `127.0.0.1` everywhere, never `localhost` or `0.0.0.0` (avoids DNS resolution and macOS firewall prompts)
- The warm-up response is raw audio bytes — just `io.Copy(io.Discard, resp.Body)` to drain it
